# Interaction System

언리얼 엔진에서 **플레이어와 오브젝트 간의 상호작용(Interaction)** 을 구현하고 학습한 내용을 정리한 레포지토리입니다.  
C++ 기반의 상호작용 시스템 제작 과정을 단계별로 기록했습니다.
각 기능을 브랜치 단위로 나누어 단계별로 구현했습니다.

---

##  프로젝트 개요

- **프로젝트명:** Interaction System  
- **엔진 버전:** Unreal Engine 5.6 
- **언어:** C++ / Blueprint  
- **학습 목적:**  
  - C++로 인터랙션 로직 구조 이해

---
##  자동문 (DoorBase)

**브랜치:** `Door`  
### 개요
플레이어가 문 앞에 다가가면 자동으로 열리고, 일정 시간 후 닫히는 시스템을 C++로 구현했습니다.  
언리얼 블루프린트의 `OnComponentBeginOverlap` / `OnComponentEndOverlap` 이벤트를 C++로 옮겨  
보다 단순하고 효율적으로 동작하도록 구성했습니다.

### 주요 기능
- `OverlapBox`로 플레이어 감지  
- `CheckDir()` 함수로 문 정면/후면 방향 판별  
- `FInterpTo`를 사용해 문이 부드럽게 회전하도록 구현  
- 플레이어가 영역을 벗어나면 **3초 후 자동으로 닫힘**  
- 타임라인, 커브 없이 Tick 기반으로 간소화된 동작

##  조명 제어 (AAutoLightController)

**브랜치:** `Lighting`  
### 개요
시간대에 따라 자동으로 조명의 밝기를 조정하는 액터를 C++로 구현한 예제입니다.  
`if` / `return` 문을 활용한 조건 분기와, 함수의 매개변수·반환값 개념을 학습하는 것이 핵심입니다.  
또한 `UFUNCTION()`으로 블루프린트에서도 호출 가능한 구조로 설계했습니다.

### 주요 기능

- `UPointLightComponent`를 이용해 **조명 액터** 생성  
- `GetBrightnessByTime(int Hour)` 함수로 **시간대별 밝기 계산**  
- **if / return** 문을 활용한 조건 분기 학습  
- `UFUNCTION(BlueprintCallable)`을 사용해 **블루프린트에서도 호출 가능**  
- `BeginPlay()`에서 임의의 시간 지정 후 조명 밝기 자동 적용  
- `UE_LOG`로 콘솔에 시간 및 밝기 로그 출력  
- 시간대별 밝기 설정:
  | 시간대 | 상태 | 밝기 (루멘) |
  | --- | --- | --- |
  | 0~6시 | 새벽 (어두움) | 200 |
  | 6~12시 | 아침 (보통 밝기) | 1500 |
  | 12~18시 | 낮 (가장 밝음) | 5000 |
  | 18~24시 | 저녁 (약간 어두움) | 800 |

  #  Auto Curtain Controller (AAutoCurtainController)

**브랜치:** `Curtain`  
### 개요

풍속(`WindSpeed`)과 온도(`Celsius`)에 따라 **자동으로 커튼 개방 정도를 조정하는 시스템**을 구현한 연습 예제입니다.  
`if` / `return` 문으로 조건 분기 구조를 익히고 함수 반환값을 이용해 메시 회전(개방 비율)을 제어합니다.

### 목표

- 조건 분기문 (`if` / `return`) 이해  
- 매개변수가 2개 이상인 함수 구조 설계  
- `float` 반환값을 이용한 비율 계산  
- 결과를 `UStaticMeshComponent`의 회전에 적용

### 주요 기능

| 조건 | 개방 비율(`Ratio`) |
| --- | --- |
| 온도 0-15°C, 풍속 0-5m/s | 1.0 (완전 개방) |
| 온도 15-25°C, 풍속 5-10m/s | 0.5 (절반 개방) |
| 온도 25°C 이상 또는 풍속 10m/s 이상 | 0.2 (거의 닫힘) |
